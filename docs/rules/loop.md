# AI 作業フロー

## 1. Analyze

- 依頼内容、完了条件、変更範囲を確認する。
- `docs/status.md` で現在状況を確認する。

## 2. Search

- 関連ファイルと類似実装を探す。
- 既存の設計、命名、データ構造を確認する。

## 3. Plan

- 変更対象と確認方法を整理する。
- ドキュメント更新が必要か判断する。

## 4. Implement

- 既存パターンに合わせて小さく実装する。
- 不要な抽象化や依頼外の変更を避ける。

## 5. Test

- 可能な範囲でテスト、ビルド、表示確認を行う。

## 6. Review

- 差分を見直し、依頼範囲外の変更やリンク切れがないか確認する。

## 7. Complete

- 変更内容、確認結果、残タスクを報告する。
